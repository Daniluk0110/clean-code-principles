# SOLID — Все вместе (Комбайн)

**Бизнес‑задача:** система импорта товаров из разных источников (XML, JSON) с валидацией, сохранением в хранилище и уведомлением администратора.

## Как соблюдаются принципы SOLID
- **S (Single Responsibility):** каждый класс делает ровно одну вещь: парсер читает данные, валидатор проверяет, репозиторий сохраняет, нотификатор уведомляет, сервис координирует процесс.
- **O (Open/Closed):** чтобы добавить новый формат импорта (CSV) или новый канал уведомлений (Slack), мы создаем новые классы, не изменяя сервис импорта.
- **L (Liskov Substitution):** любой `ProductParser` можно подставить вместо другого, и сервис продолжит работать корректно.
- **I (Interface Segregation):** интерфейсы маленькие и конкретные: `ProductParser`, `ProductValidator`, `ProductRepository`, `Notifier`.
- **D (Dependency Inversion):** `ProductImportService` зависит от абстракций, конкретные реализации передаются снаружи.
